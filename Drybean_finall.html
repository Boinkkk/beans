PENJELASAN SETIAP PENGUKURAN#

  1. Jumlah (Count): Ini menunjukkan berapa banyak entri data yang ada untuk setiap variabel. Dalam kasus ini, Anda memiliki 900 entri untuk setiap variabel.

  2. Rata-rata (Mean): Ini adalah nilai rata-rata dari seluruh data untuk setiap variabel. Rata-rata memberi Anda gambaran tentang nilai tengah dari distribusi data.

  3. Standar Deviasi (Standard Deviation): Standar deviasi mengukur seberapa jauh titik data tersebar dari rata-rata. Semakin tinggi nilai standar deviasi, semakin besar variasi dalam data.

  4. Minimum: Ini adalah nilai terkecil yang ada dalam dataset untuk setiap variabel. Ini memberi Anda gambaran tentang rentang nilai yang mungkin untuk setiap variabel.

  5. Persentil ke-25 (Q1): Ini adalah nilai yang membagi 25% data terendah dari yang lainnya. Dalam konteks ini, 25% dari data memiliki nilai di bawah nilai ini.

  6. Persentil ke-50 (Q2): Ini adalah nilai yang membagi 50% data terendah dari yang lainnya. Dalam konteks ini, 50% dari data memiliki nilai di bawah nilai ini.

  7. Persentil ke-75 (Q3): Ini adalah nilai yang membagi 75% data terendah dari yang lainnya. Dalam konteks ini, 75% dari data memiliki nilai di bawah nilai ini.

  8. Maximum: Ini adalah nilai terbesar yang ada dalam dataset untuk setiap variabel.

    if isinstance(data, pd.DataFrame) is False:
        return print("Data yang di input bukan dataframe")
    else:
        is_missing = data.isnull().values.any()
        total_missing = data.isnull().sum().sum()
        df = {
            'Missing Value?' : [is_missing],
            'Total' : [total_missing]
        }
        return pd.DataFrame(df, index=['Result'])

PEMERIKSAAN OUTLIER#

Deteksi outlier adalah metode untuk mengidentifikasi titik-titik data yang dianggap sebagai outlier atau anomali. Outlier adalah nilai yang secara signifikan berbeda dari sebagian besar data lainnya dalam sebuah dataset. Mengidentifikasi outliers bermanfaat untuk meningkatkan akurasi analisis dan model prediktif, pada kali akan menggunakan euclidian distance

Pada Kasus dataset kali ini kita akan menghapus hanya 1 nilai saja dikarenakan anomaly score yang terlalu tinggi sehingga kemungkinan besar data itu mengalami error saat ektraksi fitur. kita tidak melakukan penghapus terhadap nilai yang “dianggap outlier” dikarenakan hal itu tidak diperlukan, data yang beranomaly score tinggi sebetulnya hanyalah variasi dari Dry Bean tersebut layaknya manusia yang membunyai berat badan berlebih dan ada juga yang kekurangan. pengapusan outlier juga membuat ruang lingkup belajar model lebih sedikit sehingga rawan mengalami overfitting

def make_plot_scatter(x:list, y:list, x_title:str, y_title:str, title:str, outlier:bool=False,outlier_value = None):
    if len(x) == len(y) is False:
        return print('Jumlah Data di X dan Y tidak sama!')
    else:
        plt.style.use('fivethirtyeight')
        plt.figure(figsize=(10,6))
        plt.title(title)
        plt.xlabel(x_title)
        plt.ylabel(y_title)
        if outlier:
            plt.scatter(x, y, c=(outlier_value == -1), cmap='coolwarm', edgecolors='k')
            return plt.show()
        else:
            plt.scatter(x, y)
            plt.legend()
            return plt.show()

def get_outlier(data:pd.DataFrame, n_neighbors:int = 20,contamination:float=0.040) -> pd.DataFrame:
    data = data.select_dtypes(include='number')
    lof = LocalOutlierFactor(n_neighbors=n_neighbors, contamination=contamination, metric='euclidean')
    outlier = lof.fit_predict(data)
    outlier = pd.DataFrame(outlier)
    return outlier

def get_anomaly_scores(data:pd.DataFrame,n_neighbors:int = 20,contamination:float=0.040):
    data = data.select_dtypes(include='number')
    lof = LocalOutlierFactor(n_neighbors=n_neighbors, contamination=contamination, metric='euclidean')
    lof.fit_predict(data)
    return pd.DataFrame(lof.negative_outlier_factor_)

KNN#

Algoritma (K-NN) adalah algoritma pembelajaran mesin yang serbaguna dan banyak digunakan terutama digunakan karena kesederhanaan dan kemudahan implementasinya. Algoritma ini tidak memerlukan asumsi apa pun tentang distribusi data yang mendasarinya. Algoritma ini juga dapat menangani data numerik dan kategorikal, menjadikannya pilihan yang fleksibel untuk berbagai jenis kumpulan data dalam tugas klasifikasi dan regresi. Algoritma ini adalah metode nonparametrik yang membuat prediksi berdasarkan kesamaan titik data dalam kumpulan data tertentu. K-NN kurang sensitif terhadap outlier dibandingkan dengan algoritma lainnya.

Algoritma K-NN bekerja dengan menemukan K tetangga terdekat ke titik data tertentu berdasarkan metrik jarak, seperti jarak Euclidean. Kelas atau nilai titik data kemudian ditentukan oleh suara mayoritas atau rata-rata K tetangga. Pendekatan ini memungkinkan algoritma untuk beradaptasi dengan pola yang berbeda dan membuat prediksi berdasarkan struktur lokal data.

TREE#

Apa itu Decision Tree?#

Decision tree adalah struktur mirip diagram alir yang digunakan untuk membuat keputusan atau prediksi. Struktur ini terdiri dari node yang mewakili keputusan atau pengujian pada atribut, cabang yang mewakili hasil dari keputusan ini, dan leaf nodes yang mewakili hasil akhir atau prediksi. Setiap node internal sesuai dengan pengujian pada atribut, setiap cabang sesuai dengan hasil pengujian, dan setiap leaf node sesuai dengan label kelas atau nilai kontinu.

Struktur Decision Tree#

  • Root Node: Mewakili seluruh dataset dan keputusan awal yang harus dibuat.

  • Internal Nodes: Mewakili keputusan atau pengujian pada atribut. Setiap node internal memiliki satu atau - lebih cabang.

  • Branches: Mewakili hasil dari keputusan atau pengujian, mengarah ke node lain.

  • Leaf Nodes: Mewakili keputusan atau prediksi akhir. Tidak ada pembagian lebih lanjut pada node ini.

Bagimana Decision Tree bekerja?#

Proses pembuatan decision tree melibatkan:

  • Memilih Atribut Terbaik: Menggunakan metrik seperti Gini impurity, entropy, atau information gain, atribut terbaik untuk membagi data dipilih.

  • Membagi Dataset: Dataset dibagi menjadi subset berdasarkan atribut yang dipilih.

  • Mengulangi Proses: Proses diulangi secara rekursif untuk setiap subset, menciptakan node internal atau leaf node baru hingga kriteria penghentian terpenuhi (misalnya, semua instance dalam satu node termasuk dalam kelas yang sama atau kedalaman yang telah ditentukan tercapai).

Metrics untuk membagi#

  • Gini Impurity: Mengukur kemungkinan klasifikasi yang salah dari instance baru jika diklasifikasikan secara acak menurut distribusi kelas dalam dataset.

  • Entropy: Mengukur jumlah ketidakpastian atau impurity dalam dataset

  • Information Gain: Mengukur pengurangan entropy atau Gini impurity setelah dataset dibagi berdasarkan atribut.

Keuntungan Decision Trees#

  • Kesederhanaan dan Interpretabilitas: Decision trees mudah dipahami dan diinterpretasikan. Representasi visualnya sangat mirip dengan proses pengambilan keputusan manusia.

  • Fleksibilitas: Dapat digunakan untuk tugas klasifikasi dan regresi.

  • Tidak Memerlukan Scaling Fitur: Decision trees tidak memerlukan normalisasi atau scaling data.

  • Menangani Hubungan Non-linear: Mampu menangkap hubungan non-linear antara fitur dan variabel target.

Kekurangan Decision Trees#

  • Overfitting: Decision trees dapat dengan mudah overfit data pelatihan, terutama jika mereka dalam dengan banyak node.

  • Ketidakstabilan: Variasi kecil dalam data dapat menghasilkan tree yang sangat berbeda.

  • Bias terhadap Fitur dengan Level Lebih Banyak: Fitur dengan level lebih banyak dapat mendominasi struktur tree.

Apa itu SVM (Support Vector Machine)?#

Support Vector Machine (SVM) adalah algoritma pembelajaran mesin yang digunakan untuk klasifikasi dan regresi. SVM bekerja dengan menemukan hyperplane di ruang dimensi tinggi yang secara jelas memisahkan titik data dari kelas yang berbeda. Tujuan utama dari SVM adalah untuk menemukan hyperplane yang memiliki margin terbesar (jarak terjauh) antara titik data dari kedua kelas.

Bagaimana SVM Bekerja?#

  • Hyperplane: Dalam konteks SVM, hyperplane adalah garis atau permukaan yang memisahkan data menjadi kelas yang berbeda. Dalam dua dimensi, ini adalah garis; dalam tiga dimensi, ini adalah bidang.

  • Support Vectors: Titik data yang paling dekat dengan hyperplane disebut support vectors. Mereka adalah elemen penting dari dataset karena mereka menentukan posisi dan orientasi hyperplane.

  • Margin: Jarak antara hyperplane dan support vectors disebut margin. SVM mencoba memaksimalkan margin ini untuk memastikan bahwa hyperplane memisahkan kelas dengan baik.

Jenis-Jenis Kernel dalam SVM#

Kernel adalah fungsi yang mengubah data ke dalam bentuk yang lebih tinggi dari dimensi yang membuatnya lebih mudah untuk memisahkan dengan hyperplane. Beberapa jenis kernel yang umum digunakan adalah:

  • Linear Kernel: Digunakan saat data dapat dipisahkan secara linear. Fungsi kernel linear adalah: \(K(x_i, x_j) = x_i \cdot x_j\)

  • Polynomial Kernel: Digunakan saat data tidak dapat dipisahkan secara linear. Fungsi kernel polynomial adalah: \(K(x_i, x_j) = (\gamma x_i \cdot x_j + r)^d\)

  • Radial Basis Function (RBF): Digunakan saat data tidak dapat dipisahkan secara linear. Fungsi kernel RBF adalah: \(K(x_i, x_j) = \exp(-\gamma ||x_i - x_j||^2)\)

  • Sigmoid Kernel: Digunakan saat data tidak dapat dipisahkan secara linear. Fungsi kernel sigmoid adalah: \(K(x_i, x_j) = \tanh(\gamma x_i \cdot x_j + r)\)

Kelebihan SVM#

  • Efektif dalam Dimensi Tinggi: SVM tetap efektif dalam ruang berdimensi tinggi dan bahkan dalam kasus di mana jumlah dimensi lebih besar dari jumlah sampel.

  • Memori Efisien: SVM menggunakan subset dari titik pelatihan (support vectors) dalam proses keputusan, yang membuatnya memori efisien.

  • Flexibilitas dengan Kernel: SVM dapat menggunakan berbagai fungsi kernel untuk memodelkan hubungan non-linear antara fitur.

Kekurangan SVM#

  • Waktu Pelatihan yang Lama: SVM bisa lambat dalam pelatihan, terutama untuk dataset yang sangat besar.

  • Pemilihan Kernel dan Parameter: Memilih kernel yang tepat dan mengatur parameter SVM (seperti C dan 𝛾) dapat menjadi sulit dan memerlukan cross-validation.

  • Kurang Efektif pada Dataset Besar dengan Noise: SVM kurang efektif jika terdapat banyak noise dan outliers dalam data karena mereka dapat mempengaruhi posisi hyperplane.

ENSEMBLE LEARNING#

Setelah dilakukan Pemodelan, ada banyak cara untuk meningkatkan akurasi prediksi dari model yang telah kita buat salah satunya adalah Ensemble Learning.

Apa itu Ensemble Learning?

PENGERTIAN ENSEMBLE LEARNING#

Ensemble learning adalah teknik yang digunakan untuk meningkatkan kinerja dan akurasi model-model pembelajaran mesin dengan menggabungkan hasil prediksi dari beberapa model atau algoritma yang berbeda.

Istilah ensemble merujuk pada konsep menggabungkan banyak elemen menjadi satu kesatuan. Konsep penggabungan berbagai pendekatan pembelajaran mesin diharapkan dapat mengatasi kelemahan individu dari masing-masing model dan menciptakan prediksi yang lebih akurat dan stabil.

Cara kerja dari metode ensemble melibatkan penggunaan beberapa model yang beragam dalam satu tim. Hasil prediksi dari masing-masing model tersebut digabungkan untuk membuat keputusan akhir.

Pada project kali ini kita akan menggunakan 3 metode dalam ensemble learning yaitu Bagging, Random Forest, dan Stakcing dan mencari metode mana yang terbaik untuk data kita.

BAGGING#

PENGERTIAN METODE BAGGING#

Bagging (bootstrap aggregating) merupakan metode ensemble yang melibatkan pelatihan beberapa model secara sendiri pada dataset yang diacak, dan menggabungkan prediksi mereka melalui pemungutan suara.

setiap model dilatih pada dataset yang diacak dari data yang diambil sampelnya,  data individual dapat dipilih lebih dari satu kali. data acak ini dikenal sebagai sampel bootstrap. Dengan melatih model pada bootstrap yang berbeda, bagging mengurangi varians dari masing-masing model. Bagging juga dapat menghindari overfitting.

Prediksi dari semua model yang diambil sampelnya kemudian digabungkan dan dibuat voting yang dimana hasil prediksi terbanyak akan di keluarkan sebagai hasil prediksi akhir

Bagging sangat efektif dalam mengurangi varians dan overfitting, membuat model lebih tangguh dan akurat, terutama dalam kasus di mana masing-masing model rentan terhadap variabilitas tinggi.

Dan pada percobaan kali ini kita akan melakukan proses bagging kepada setiap algortima terbaik dari setiap model yang telah kita hitung diatas. dan model terbaik antara model normal atau model bagging akan dipilih yang terbaik dan akan dimasukkan pada metode stacking yang akan kami jelaskan nanti dan semua perhitungan disin menggunakan data yang di standarisasi dikarenakan data tersebut yang memiliki akurasi paling tinggi, masing masing iterasi akan mengubah nilai n_estimatorsnya(juri) dari 3 sampai 49 dengan step 2

RANDOM FOREST#

Apa itu Random Forest?#

Random Forest adalah algoritma pembelajaran ensemble yang menggunakan beberapa decision tree untuk meningkatkan performa prediksi dan mengurangi overfitting. Ini adalah salah satu metode paling populer dan kuat dalam pembelajaran mesin, baik untuk tugas klasifikasi maupun regresi.

Bagaimana Random Forest Bekerja?#

  1. Pengambilan Sampel Acak (Bootstrapping): Algoritma memulai dengan pengambilan sampel acak dengan penggantian (bootstrapping) dari dataset pelatihan asli untuk membuat beberapa subset data.

  2. Membangun Decision Tree: Untuk setiap subset data, sebuah decision tree dibangun. Saat membangun setiap tree, hanya subset acak dari fitur yang dipertimbangkan untuk pembagian di setiap node (feature randomness). Ini membantu dalam menciptakan variasi di antara tree yang berbeda.

  3. Prediksi Ensemble: Setelah semua decision tree dibangun, prediksi dilakukan dengan menggabungkan hasil dari semua tree. Untuk klasifikasi, prediksi akhir adalah kelas yang dipilih oleh mayoritas tree (voting mayoritas). Untuk regresi, prediksi akhir adalah rata-rata dari semua prediksi tree.

Keuntungan Random Forest#

  • Mengurangi Overfitting: Dengan menggabungkan hasil dari banyak decision tree, Random Forest mengurangi risiko overfitting yang sering terjadi pada single decision tree.

  • Robustness: Tahan terhadap outlier dan data yang hilang.

  • Flexibilitas: Dapat digunakan untuk tugas klasifikasi dan regresi.

  • Feature Importance: Dapat memberikan informasi tentang pentingnya fitur (feature importance), membantu dalam memilih fitur yang relevan.

Kekurangan Random Forest#

  • Kompleksitas dan Waktu Komputasi: Membutuhkan lebih banyak sumber daya komputasi dan waktu untuk melatih dan membuat prediksi dibandingkan dengan decision tree tunggal.

  • Kurang Interpretabilitas: Meskipun lebih akurat, hasil dari Random Forest lebih sulit untuk diinterpretasikan dibandingkan dengan single decision tree.

Langkah-langkah dalam Random Forest:#

  1. Membangun Forest:

    • Mengambil sampel acak dengan penggantian dari dataset asli untuk membuat beberapa subset.

    • Membangun decision tree untuk setiap subset dengan mempertimbangkan subset acak dari fitur untuk pembagian di setiap node.

  2. Prediksi:

    • Untuk klasifikasi: Voting mayoritas dari prediksi semua tree.

    • Untuk regresi: Rata-rata dari semua prediksi tree.

STACKING#

PENGERTIAN METODE STACKING#

    Stacking adalah sebuah metode dimana sebuah model ditraining untuk menggabungkan beberapa individual model yang disebut first-level learner, sedangkan yang digabungkan disebut second-level learner atau meta-learner.

    Stacking melibatkan dua atau lebih base learner atau model dasar sebagai model level-1, dan meta-learner yang menggabungkan prediksi base learner disebut sebagai level-2. Model yang berbeda digunakan oleh base learner untuk belajar dari suatu dataset. Output dari masing-masing model base learner dikumpulkan untuk membuat dataset baru. Di dalam dataset yang baru, setiap sampel berhubungan dengan nilai pada dataset awal yang seharusnya diprediksi. Selanjutnya dataset tersebut digunakan oleh model meta-learner hingga mendapatkan hasil akhir.

Deskripsi :

  1. First-level Learner

  2. Data set baru yang dihasilkan dari hasil pembelajaran first-level Learner

  3. Meta Learner

  4. Hasil Prediksi
